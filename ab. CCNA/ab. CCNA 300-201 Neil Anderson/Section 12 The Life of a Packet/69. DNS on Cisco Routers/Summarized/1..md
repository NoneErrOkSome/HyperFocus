### üéØ **Goal: Configure a Cisco Router as a DNS Server and Another as a DNS Client to Resolve Hostnames to IP Addresses**

This setup allows one router to act like a DNS server (similar to a public DNS server like Google‚Äôs 8.8.8.8) and other routers to query it, resolving hostnames (like `R2.flackbox.lab`) into IP addresses (`10.10.10.2`), enabling easier and more human-readable communication within the network.

#### üîπ On **Router R3** (acts as the **DNS Server**):

> This router stores a list of hostname-to-IP mappings and answers DNS queries from other routers.

1. **Enable DNS lookup (default is enabled, but good practice to ensure):**
    
    ```bash
    configure terminal
    ip domain-lookup
    ```
    
    ‚úîÔ∏è _Allows the router to resolve hostnames into IP addresses via DNS._
    
2. **Point DNS to itself:**
    
    ```bash
    ip name-server 10.10.20.1
    ```
    
    ‚úîÔ∏è _R3 uses its own IP (acting as DNS server). Required for self-resolution in some cases (e.g., testing)._
    
3. **Set internal domain name:**
    
    ```bash
    ip domain-name flackbox.lab
    ```
    
    ‚úîÔ∏è _Adds suffix automatically when incomplete hostnames are entered. Also used in full FQDN entries like `R1.flackbox.lab`._
    
4. **Enable DNS server functionality:**
    
    ```bash
    ip dns server
    ```
    
    ‚úîÔ∏è _Makes R3 an authoritative DNS server that can respond to queries._
    
5. **Add hostname-to-IP mappings:**
    
    ```bash
    ip host R1 10.10.10.1
    ip host R2 10.10.10.2
    ip host R3 10.10.20.1
    ip host R1.flackbox.lab 10.10.10.1
    ip host R2.flackbox.lab 10.10.10.2
    ip host R3.flackbox.lab 10.10.20.1
    ```
    
    ‚úîÔ∏è _Maps both short and fully qualified domain names (FQDN) to IP addresses. This allows resolution of names like `R1` or `R1.flackbox.lab`._
    

---

#### üîπ On **Router R1** (acts as the **DNS Client**):

> R1 is set up to send queries to R3 when trying to resolve hostnames.

1. **Enable DNS lookup:**
    
    ```bash
    configure terminal
    ip domain-lookup
    ```
    
    ‚úîÔ∏è _Enables DNS resolution on R1 when hostnames are entered (e.g., in `ping R2`)._
    
2. **Specify R3 as DNS server:**
    
    ```bash
    ip name-server 10.10.20.1
    ```
    
    ‚úîÔ∏è _Directs R1‚Äôs DNS queries to R3._
    
3. **Add domain suffix for hostname completion:**
    
    ```bash
    ip domain-list flackbox.lab
    ```
    
    ‚úîÔ∏è _If R1 tries to resolve `R2`, it automatically appends `.flackbox.lab` making it `R2.flackbox.lab`._

### üß™ **Test Explanation**

    ```bash

    ```
    
    - If `R2` is not found in local cache, R1 sends a DNS query to R3.
        
    - R3 responds with `10.10.10.2`.
        
    - R1 now knows the IP and sends ICMP Echo Requests (ping) to it.
        
- ‚úÖ If ping works, DNS resolution is successful.
